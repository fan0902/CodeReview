# CR Branch Review 风格与文件目录过滤设计

## 1. 目标

在不改变 CR 现有本地只读分析、代码跳转、Controller 与 Enum 功能的前提下：

1. 让浏览器界面在视觉语言上对齐用户提供的 Branch Review 参考图。
2. 保持界面简洁、信息层级明确，并优先保证代码阅读区的可读性。
3. 在左侧边栏增加常驻的文件与目录关键词过滤。
4. 让关键词过滤与现有“显示隐藏文件”开关使用同一棵派生文件树。

本次对齐的是配色、间距、边框、圆角、按钮、卡片和区域层级，不引入参考图中的分支选择、提交关系或差异统计功能。

## 2. 已确认交互

- 过滤输入同时匹配节点名称和工程内完整相对路径。
- 匹配不区分大小写，并忽略输入首尾空白。
- 子文件或子目录命中时，保留从工程根到命中项的父目录层级。
- 目录自身命中时，展示该目录的全部可见子树。
- “显示隐藏文件”关闭时，先递归移除点文件和点目录，再执行关键词过滤。
- “显示隐藏文件”打开时，关键词可以命中服务端文件树中存在的隐藏项。
- 永久忽略的 `.git`、`node_modules`、虚拟环境、缓存和构建产物仍由服务端排除，不受前端开关或过滤词影响。
- 清空关键词后恢复当前隐藏文件设置下的完整文件树。

## 3. 方案选择

采用“统一视觉 token + 局部组件结构调整 + 前端纯函数过滤”方案。

未采用以下方案：

- 仅修改 CSS：开发量较小，但顶部、左侧工具区和文件节点仍缺少参考图中的信息层级与卡片感。
- 完整复刻参考图：会引入 CR 不需要的分支比较工具和双栏差异语义，增加无关界面与维护成本。
- 服务端关键词查询：现有文件树已经一次性加载到前端，本地过滤响应更快，也无需扩展接口或增加索引请求。

## 4. 视觉设计

### 4.1 主题 token

浅色主题使用暖白画布、米白侧栏、暖灰边框和深黑主要操作，替换当前偏冷的蓝灰视觉：

- 页面画布：暖白色。
- 主要内容面：接近白色，但保留轻微暖色调。
- 次级区域和侧栏：浅米色，形成柔和分区。
- 边框：低对比暖灰，使用 1px 细线。
- 主要文字：近黑色；路径、描述和辅助状态使用中性暖灰。
- 主要按钮：近黑背景、白色文字。
- 状态强调：低饱和绿色、蓝色和砖红色，分别用于正常、焦点与危险动作。
- 圆角：输入框和按钮约 10px，信息卡片约 12px。
- 阴影：只在浮层或必要的选中层级使用轻微阴影，不给所有区域增加阴影。

现有暗色模式继续保留，但采用相同的层级关系和组件形状；视觉验收以参考图对应的浅色模式为主。

### 4.2 顶部工具区

顶部从当前紧凑单行工具条调整为清晰的应用级标题区：

- 左侧显示 CR 标识、应用名和本地只读说明。
- 中部将当前工程绝对路径放入输入框式只读容器，长路径单行省略，悬停仍可查看完整路径。
- “打开工程”使用近黑色主按钮，成为唯一高强调操作。
- 索引状态显示为轻量状态胶囊；就绪状态使用低饱和绿色提示。
- 未打开工程时，路径容器展示引导文案，不伪造本地路径。

顶部不会增加分支选择器或差异比较按钮。

### 4.3 左侧文件边栏

左侧使用比主代码区略深的米色背景，结构分为：

1. 工程摘要：工程图标、名称和弱化的绝对路径。
2. 文件工具：关键词过滤框、清空操作、快速打开入口和隐藏文件开关。
3. 文件树：可滚动的目录和文件层级。

过滤框常驻显示，标签或占位文案为“过滤文件或目录”。清空按钮仅在存在关键词时出现，并提供可访问名称。

目录保留原生可折叠语义。文件节点增加轻量的语言类型标识：Python 使用低饱和蓝色，TypeScript 使用低饱和蓝紫色，其他文件使用中性标识。当前打开文件使用清晰描边与暖白背景，悬停只改变背景，不产生布局位移。

文件名为主信息，相对目录为辅助信息；目录层级和缩进仍是结构的首要表达，不把树改造成脱离层级的平铺搜索结果。

### 4.4 中间代码区

- Monaco 编辑器继续作为代码阅读主体，不修改语法高亮、定义跳转或导航历史行为。
- 文件标签栏使用暖灰背景、细边框和更明确的选中标签。
- 编辑器外围减少无意义装饰，保证最大可用宽度与高度。
- 欢迎页和空状态使用与新主题一致的标题、辅助说明和最近工程卡片。

### 4.5 右侧信息区

- Controllers 与 Enums 标签改为简洁的分段标签样式，选中项使用深色文字和低饱和强调线。
- 筛选输入、下拉框、Endpoint 卡片和 Enum 卡片统一边框、圆角、间距与文字层级。
- HTTP 方法继续保留语义色，但降低饱和度，避免与代码内容争夺注意力。
- 折叠按钮使用清晰的窄栏状态，不改变现有展开与折叠功能。

## 5. 文件树过滤数据流

文件树派生分为两个独立纯函数步骤：

1. `visibleFileTree` 根据 `showHidden` 生成允许展示的基础树。
2. 新的关键词过滤函数接收基础树与查询词，生成最终展示树。

过滤函数遵循以下递归规则：

- 查询为空时原样返回基础树，避免无意义复制。
- 文件名称或相对路径命中时保留文件。
- 目录名称或相对路径命中时保留目录及其完整可见子树。
- 目录本身未命中、但有后代命中时，保留目录和过滤后的命中后代。
- 目录和后代均未命中时移除该目录。
- 不修改 React Query 保存的原始服务端结果，也不改变节点顺序。

最终树同时提供给可见文件树和 `QuickOpen`。这样隐藏项语义保持一致；常驻筛选只约束左侧树，`QuickOpen` 仍可在当前可见基础树中执行自己的临时搜索，避免两个输入框互相覆盖状态。

## 6. 状态与空状态

- `FileBrowser` 持有 `showHidden` 和 `filterQuery` 两个页面内状态。
- 切换工程时清空过滤词，避免旧工程的查询造成新工程看似没有文件。
- 隐藏文件开关保持现有非持久化语义。
- 有关键词但最终树为空时，展示“没有匹配的文件或目录”，并提供清空筛选操作。
- 文件树请求加载、失败和未打开工程时继续使用各自独立状态，不显示误导性的过滤空状态。

## 7. 响应式与可访问性

- 1440px 宽度下保持完整三栏布局，并让左栏有足够宽度展示文件名和辅助路径。
- 1024px 宽度下继续将右侧信息区收起为窄栏，代码区宽度不低于现有 560px 可读性下限。
- 顶部路径区域允许收缩和省略，主要按钮与状态不会被挤出视口。
- 过滤框具备可访问标签；清空按钮、隐藏文件开关、文件树和折叠按钮保持明确名称。
- 所有交互元素保留可见键盘焦点，颜色对比不作为唯一状态表达。
- `prefers-reduced-motion` 下不添加非必要动画。

## 8. 组件与文件边界

预计修改范围：

- `ProjectToolbar`：调整标题区结构和状态展示，不改变工程选择或索引请求。
- `FileBrowser`：增加过滤状态、工具区和空状态，并标识当前文件。
- 文件树过滤纯函数：增加关键词递归过滤能力。
- `FileTabs`、`InformationPanel` 与现有信息卡片：仅增加必要的结构类名或辅助标识。
- `tokens.css` 与 `layout.css`：落地主题、组件层级和响应式样式。
- Web 单元测试与 Playwright 测试：覆盖过滤语义、布局和视觉回归。

不修改服务端接口、分析器、导航协议、本地设置格式、Swift 启动器或 macOS App 生命周期。

## 9. 测试策略

严格按测试先行实施：

### 9.1 纯函数测试

- 空查询返回基础树。
- 文件名命中时保留祖先目录。
- 完整相对路径命中时保留祖先目录。
- 目录自身命中时保留完整可见子树。
- 不区分大小写并忽略首尾空白。
- 无匹配时返回空树。
- 隐藏过滤与关键词过滤按确定顺序组合。

### 9.2 组件测试

- 左侧展示“过滤文件或目录”输入框。
- 输入关键词后只展示匹配节点及其祖先。
- 清空按钮恢复文件树。
- 无匹配时展示空状态。
- 切换工程时清空查询。
- 当前文件节点获得可访问的选中状态。
- 现有隐藏文件和 Quick Open 行为不回归。

### 9.3 端到端与视觉验收

- 在混合 Python/TypeScript 工程中按文件名与目录路径过滤并打开文件。
- 验证 Controller、Enum、定义跳转和标签页行为不回归。
- 在 1440×900 与 1024×768 下确认无横向溢出、代码区宽度合格、键盘焦点可见。
- 生成浅色与暗色截图；浅色模式逐项对照参考图的暖色主题、顶部层级、左侧卡片感和右侧信息卡片。

### 9.4 发布验证

- 运行全部 JS/TS 单元测试、类型检查和生产构建。
- 运行 Playwright 测试及截图检查。
- 运行 Swift 测试、macOS bundle 测试、仓库卫生检查和 smoke 测试。
- 重新构建、签名并启动 `CR.app`，在系统默认浏览器做最终人工视觉检查。

## 10. 验收标准

- 整体视觉明显对齐参考图的暖白、米色、深黑按钮、细边框、圆角卡片与低饱和状态色。
- 界面仍然是 CR 代码阅读器，没有出现无关的分支比较功能。
- 顶部可清晰识别应用、当前工程绝对路径、打开工程操作和索引状态。
- 左侧能够按文件名、目录名或相对路径实时过滤，并保持可理解的父目录层级。
- 隐藏文件设置与关键词过滤组合结果一致，永久忽略目录不会被恢复。
- Python、TypeScript 代码、跳转、Controller、Enum 和本地保存删除功能保持可用。
- 1024px 和 1440px 两档无横向溢出，代码区保持可读，键盘焦点清晰。
- 仓库中不出现开发机绝对路径、用户名或其他旧品牌文本。
