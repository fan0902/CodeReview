# CR macOS App 图标设计规格

## 1. 目标

把当前 `CR.app` 的默认应用图标替换为用户截图中由红色矩形边框和蓝色粗体 `CR` 构成的标识。图标必须在 Finder、Dock 和启动台中保持简洁、清楚，并与应用名称 CR 一致。

## 2. 已批准的视觉方案

采用已确认的方案 B：

- 使用 macOS 风格的白色圆角方形底板。
- 在底板中央放置红色横向矩形边框。
- 红框内只显示蓝色粗体大写文字 `CR`。
- 去除截图中的“未打开工程”、分隔线和其他界面内容。
- 标识四周保留足够安全边距，避免 Finder 和 Dock 的小尺寸裁切或粘边。
- 不增加渐变、阴影堆叠、图案、附加文字或其他装饰。

文字内容必须严格为 `CR`，不能出现相似字形、额外字符或错误大小写。

## 3. 图像与资源规格

主源图为 1024 × 1024 像素的正方形 PNG，采用透明画布承载白色圆角底板。圆角底板、红框和 `CR` 均水平、垂直居中。红框采用醒目的纯红色，文字采用高对比度的深蓝色；最终颜色以视觉接近用户截图且在小尺寸下清晰为准。

主源图用于生成标准 macOS iconset：

- 16 × 16 与 32 × 32（`16x16` 和 `16x16@2x`）。
- 32 × 32 与 64 × 64（`32x32` 和 `32x32@2x`）。
- 128 × 128 与 256 × 256（`128x128` 和 `128x128@2x`）。
- 256 × 256 与 512 × 512（`256x256` 和 `256x256@2x`）。
- 512 × 512 与 1024 × 1024（`512x512` 和 `512x512@2x`）。

iconset 由系统 `iconutil` 转换为 `resources/CR.icns`。1024 × 1024 PNG 同时保留在 `resources/CR-icon-1024.png`，作为可复现的设计源文件。

## 4. 生成和质量控制

使用用户截图作为视觉参考，通过内置图像生成能力创建白色圆角底板版本。生成结果进入项目之前必须人工检查：

- 文字准确为 `CR`。
- 红框完整、线条清晰、未贴近底板边缘。
- 没有“未打开工程”、水印或其他多余内容。
- 1024 × 1024 原图边缘干净，没有明显生成伪影。
- 缩小到 16、32、64 和 128 像素后仍能辨认红框及 `CR`。

若生成图中的字形不准确，可以在生成结果基础上进行确定性的本地排版修正，以保证品牌文字准确；修正不得改变已经批准的视觉结构。

## 5. macOS 包集成

`resources/Info.plist.in` 增加 `CFBundleIconFile`，值为 `CR`。`scripts/build-macos-app.sh` 在签名前把 `resources/CR.icns` 复制到 `CR.app/Contents/Resources/CR.icns`。

`scripts/test-macos-bundle.sh` 增加以下断言：

- `Contents/Resources/CR.icns` 存在且非空。
- 产物 `Info.plist` 的 `CFBundleIconFile` 为 `CR`。
- 现有包结构检查和严格签名校验继续通过。

重新构建后，ad-hoc 签名覆盖包含新图标资源的完整 App。最终产物仍交付到 `outputs/CR.app`，不改变应用名称、Bundle Identifier、启动方式或业务功能。

## 6. 验收标准

完成必须同时满足：

1. Finder、Dock 或启动台显示白色圆角底板、红框、蓝色 `CR` 的新图标。
2. 图标没有截图中的其他文字或界面元素，小尺寸下仍可辨认。
3. `CR.app/Contents/Info.plist` 正确声明 `CFBundleIconFile=CR`。
4. `CR.app/Contents/Resources/CR.icns` 存在，并包含标准 macOS 图标尺寸。
5. 项目测试、类型检查、前端和服务构建、Swift 启动器构建、macOS 包测试、启动 smoke test 与 `codesign --verify --deep --strict` 均通过。
6. 新产物覆盖共享交付位置后再次通过签名和包结构校验。

## 7. 范围外事项

本次不修改 CR 浏览器界面中的 Logo、配色或布局，不改变 App 名称和版本号，不引入动态图标、菜单栏图标或多套主题图标。
